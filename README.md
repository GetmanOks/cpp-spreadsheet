# C++ Spreadsheet Engine

[![C++ CI](https://github.com/GetmanOks/cpp-spreadsheet/actions/workflows/ci.yml/badge.svg)](https://github.com/GetmanOks/cpp-spreadsheet/actions/workflows/ci.yml)

Backend-движок электронной таблицы на C++17. Поддерживает текстовые ячейки,
арифметические формулы, ссылки между ячейками и автоматический пересчёт
зависимых значений.

Проект демонстрирует работу с графом зависимостей, полиморфизмом, умными
указателями, кэшированием и генерацией парсера с помощью ANTLR.

## Возможности

- хранение текста и арифметических формул;
- ссылки на ячейки в формате `A1`, `B2`, `AA10`;
- операции `+`, `-`, `*`, `/` и скобки;
- обнаружение циклических зависимостей;
- кэширование вычисленных значений и каскадная инвалидация кэша;
- обработка ошибок `#REF!`, `#VALUE!` и `#ARITHM!`;
- вывод исходного текста и вычисленных значений таблицы;
- автоматические тесты основных и граничных сценариев.

## Как устроен проект

```text
Sheet
 └── Cell
      ├── EmptyImpl
      ├── TextImpl
      └── FormulaImpl
           ├── FormulaAST
           └── ссылки на другие Cell
```

Каждая формула хранит список используемых ячеек. Ячейки, в свою очередь,
знают о зависимых от них формулах. При изменении значения кэш
инвалидируется по графу зависимостей, а перед сохранением новой формулы
проверяется отсутствие цикла.

## Сборка и тесты

Необходимы:

- компилятор с поддержкой C++17;
- CMake 3.20 или новее;
- Java Runtime 17 или новее;
- Git и доступ в интернет при первой конфигурации.

CMake автоматически загружает ANTLR 4.13.2 и C++ runtime совместимой версии.

```bash
cmake -S spreadsheet -B build
cmake --build build --config Release
ctest --test-dir build -C Release --output-on-failure
```

При следующих сборках уже загруженные зависимости используются повторно.

## Технологии

- C++17;
- CMake;
- ANTLR 4.13.2;
- STL: `variant`, `optional`, `unique_ptr`, контейнеры и алгоритмы;
- ООП: интерфейсы, наследование и полиморфизм;
- граф зависимостей и обход в глубину;
- модульные и регрессионные тесты.

## Ограничения

Это backend без графического интерфейса и сохранения таблицы на диск.
Формулы поддерживают арифметические выражения и ссылки на ячейки, но не
содержат встроенных функций наподобие `SUM`.

## Происхождение проекта

Проект выполнен как дипломная работа курса «Разработчик C++» Яндекс
Практикума. Интерфейсы и требования были предоставлены в учебном задании;
логика таблицы, ячеек, зависимостей, кэширования и интеграция парсера
реализованы в рамках проекта.
